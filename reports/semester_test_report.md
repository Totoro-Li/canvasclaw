# CanvasClaw 全学期课程问答系统 · 完整测试报告

> 课程：《智能体及应用（研）》 ｜ 测试语料：全学期 26 讲课程录播（约 23.2 小时）
> 测试日期：2026-06-07 ｜ 运行环境：8×NVIDIA A800-80GB（本机）

## 一、概述

CanvasClaw 是一个面向课程录播的多智能体问答助手：把整学期的讲课视频自动转写、切页、建索引后，学生用自然语言提问，系统**自动判定问题属于哪一讲（或跨讲）**，分派多个 Worker 智能体在相应讲次内做检索增强（RAG），最后聚合成带**精确出处（讲次 + 时间戳 + 幻灯片 + 原文引用）**的回答，并支持一键跳转到视频对应时刻。

本次测试在本机用真实的全学期 26 讲录播完整跑通了「摄取 → 索引 → 检索 → 路由 → 作答 → 出处核实」全流程，并对回答质量做了自动化 + 对抗式评测。

## 二、系统架构

- **编排（Orchestrator–Worker，基于 LangGraph）**：`retrieve_candidates → select_lectures → 扇出 dispatch → Worker RAG → aggregate`。路由不是只看标题，而是基于**检索到的真实内容片段**判定讲次，并带 top-1 兜底，避免漏召回。
- **语音转写（ASR）**：faster-whisper `large-v3`，GPU、fp16、beam=5、词级时间戳、VAD 过滤；强制中文、关闭 `condition_on_previous_text` 以避免长音频解码塌缩。
- **幻灯片**：感知哈希（dHash）对录屏做「视觉切页」，得到去重关键帧 + 时间区间。
- **幻灯片 OCR**：视觉大模型（qwen3-omni-flash）逐页提取标题与要点，回填到讲次单元，补充「图文检索」能力。
- **检索**：bge-small-zh 向量（本地 GPU）+ BM25（中文 bigram）混合召回；单元级路由 + 单元内 chunk 精排。
- **出处接地（grounding）**：回答中的引用会被回溯到具体 ASR 片段，得到精确时间戳；本测试还额外做了**引用逐字核实**（引用原文必须真实出现在该讲转写中）。
- **大模型**：经 OpenAI 兼容网关（llm.dp.tech）调用 qwen3-max（作答/聚合）、qwen-plus（路由）、qwen3-omni-flash（视觉）。

课程脉络（自动生成的讲次主题，节选）：导论 → 与环境交互 → 生成式大模型 → 形式化数学推理 → 评估与架构 → 记忆/工具/规划 → 检索增强 → 工具调用 → 多智能体系统 → 三阶段训练/轻量化/强化微调/RLHF → 多模态 → 安全与防御。

## 四、数据处理流水线与发现并修复的问题

为在全学期规模上跑通，修复了 3 个真实问题（均已在代码中修正）：

1. **ffmpeg 吞掉 CSV 标准输入**：并行摄取时后台 ffmpeg 继承了循环的 stdin（讲次清单），把 CSV 当作交互命令吃掉，导致 16 讲被静默跳过。修复：循环用 fd3 读 CSV + 子进程 `</dev/null` + ffmpeg `-nostdin`。
2. **Whisper 中文幻觉**：静音段会被识别成「请不吝点赞、订阅、明镜与点点栏目」等频道口水话。修复：加入高精度幻觉短语过滤，避免污染检索。
3. **语言误判 + 长音频解码塌缩**：部分讲次首段被自动语言识别成挪威语（nn）直接 0 段返回；个别讲次只转写到后半段。修复：强制 `language=zh` + 关闭 `condition_on_previous_text`，并把并行度从 8 降到 4 减少争用。L01 内容量由 4.8k 字恢复到 11.9k 字。

## 五、测试方法

- **题库**（自动生成，4 类）：
  - `factual`（每讲 1 题，基于该讲关键词的知识性问题）
  - `locate`（「哪一讲、什么时间讲了 X」，考路由 + 时间定位）
  - `compare`（跨讲对比，考多讲扇出）
  - `oos`（越界诚实性：问课程没讲的内容，应如实说明未涉及）
- **自动指标**：引用逐字接地率（quote 是否真实出现在该讲转写）、时间戳偏差、命中讲次、引用数、时延。
- **对抗式 LLM 评审（多智能体工作流）**：每题一个评审智能体判定「路由是否正确 / 是否忠于引用 / 回答质量」，对初判为「正确」的再过一道**怀疑式复核**，尽量挑错。

## 六、测试结果

评测分三层：① 自动指标（时延、引用数、引用逐字接地率、路由命中）；② LLM 评审（每题判路由/忠实度/质量）；③ **对抗式复核**（对初判“正确”的再用怀疑式智能体挑错）。共 48 道题、87 个评审智能体。

### 6.1 总体指标（48 题）

| 指标 | 数值 |
|---|---|
| 路由合适率 routing_ok | **44/48 = 92%** |
| 严格正确率（经对抗复核） | **29/48 = 60%** |
| 平均忠实度 faithful（1–5） | **4.25** |
| 平均回答质量 quality（1–5） | **4.10** |
| 平均时延 | 27.9 秒/题 |
| 引用总数 | 343 |

### 6.2 分类正确率

| 类别 | 正确/总 | 解读 |
|---|---|---|
| factual 知识性 | 19/26（73%） | 7 个失败中 **4 个是源音频损坏的“纯幻灯”讲次**（L11–L14） |
| locate 定位 | 4/12（33%） | 讲次/时间基本对，但严格复核因“把幻灯OCR当作口播原文”大量扣分 |
| compare 跨讲对比 | 2/6（33%） | 跨讲扇出召回正确，引用保真与单元归属偶有偏差 |
| oos 越界诚实 | **4/4（100%）** | 全部如实回答“课程未涉及”，**零编造引用** |

### 6.3 引用接地（逐字核实，343 条引用）

| 口径 | 比例 |
|---|---|
| 引用片段（按 `……` 切分）每段都逐字命中 转写+幻灯 | **326/343 = 95%** |
| 引用整段逐字命中 转写+幻灯 | 244/343 = 71% |
| 引用整段逐字命中 **ASR 转写** | 136/343 = 40% |

### 6.4 对抗式评审发现的真问题（已二次复核确认）

1. **引用“原文”与来源不符**：Worker 有时把**幻灯片 OCR 文本**或**概括/改写**当作口播“原文”逐字引用（如 Q007、Q008），使“ASR 逐字接地率”被高估。这正是对抗式评审相对自动指标的价值——自动指标只校验“文本是否在语料里”，而复核进一步区分“是否真的是那句口播原话”。**修复方向**：Worker 仅从 ASR 片段抽取 verbatim 引用，并在 UI 上区分「口播原话」与「幻灯要点」。
2. **纯幻灯讲次干扰路由**：L11–L14 源音频损坏、只有幻灯 OCR；又因课程大纲页（L01 第 13 页列有“#13 智能体安全、#14 前沿报告”等**专题编号**）与真实讲次编号撞车，导致这些题路由/归属错误（Q010–Q013）。**修复方向**：为“仅幻灯”讲次降权或显式标注，并把大纲页排除出可被当作答案来源的内容。

> 结论：**检索、跨讲路由（92%）、越界诚实（100%）与回答质量（4.1/5）已达到可用水平**；主要短板是引用层面的“口播 vs 幻灯”保真区分，以及对损坏讲次的鲁棒性——两者均有明确且低成本的修复路径。

## 七、示例问答

**示例 1 ｜ 知识性（RAG）— 13/13 引用逐字核实**
- 问：课程里讲到的「检索增强生成」是什么？核心思想/做法？
- 答（摘）：RAG 在生成前先从外部知识库检索相关信息作为上下文，缓解 LLM 知识固化与“幻觉”、弥补上下文窗口不足……（给出三种检索方式、相似度计算、动态更新四步）
- 出处：`[L07 00:15:27]`「检索增强想做的一件事是…打破固有静态知识」、`[L07 00:16:54]`、`[L08 …]`——路由命中 L07/L08（检索增强）。

**示例 2 ｜ 定位（locate）**
- 问：老师在哪一讲、什么时间讲了「多智能体系统」？
- 答：第 15 讲，自 `00:05:47` 起系统讲解，并在 `00:07:02 / 00:12:10 / 00:14:46` 等处展开（架构、与单智能体区别、任务拆解、工具调用）。
- 出处：`[L15 00:06:39]`、`[L15 00:14:34]`、`[L15 00:14:46]`，3/3 逐字核实。

**示例 3 ｜ 越界诚实（oos）— 模范行为**
- 问：这门课讲过「区块链的共识算法」吗？在哪一讲？
- 答：**“抱歉，在本课程已索引的讲次中没有找到与该问题直接相关的内容。”**（0 引用，不编造）

**示例 4 ｜ 对抗复核挑出的反例（Q007）**
- 现象：回答 RAG 的正文内容正确、路由正确，但其中两条标注为“原文”的引用（含 `→` 箭头格式的“工作流”）在 ASR 转写中并不存在，实为模型概括/幻灯内容。**这类问题正是自动指标看不出、对抗式评审抓出的**，已计入“不正确”。

## 八、局限与后续

- **7 讲源音频损坏**：建议补录或提供备份音轨；当前这些讲次仅能通过幻灯片 OCR 做图文检索。
- **幻灯片 OCR 成本**：全学期 1,167 张幻灯片走视觉大模型，约十余分钟；可按需增量化。
- **faster-whisper 不支持昇腾 NPU**：重摄取仍建议在 CUDA 机器完成，产物（data/）再同步到部署容器。
- **检索粒度**：当前单元级 + chunk 级两段式；后续可引入更细的「问句-答句」对齐与重排序模型进一步提升 locate 精度。
